# 美术工作流指南：Figma → PSD → FairyGUI

面向美术同学。按本文做界面，程序可用工具自动转成 FairyGUI，少返工。

技术明细见同目录：[NAMING_RULES.md](./NAMING_RULES.md)

---

## 1. 这套流程在干什么？

```
需求 / 参考图
    ↓
Figma 搭界面（按命名规则）  ←── 可用 Cursor Agent + Figma MCP 协助生成
    ↓
导出 PSD（保留文字）
    ↓
psd2fgui 一键转换
    ↓
FairyGUI 导入微调（列表、动效等）
    ↓
程序接逻辑
```

| 谁来做 | 做什么 |
|--------|--------|
| 美术 | Figma 分层命名、出 PSD、可选协助转 FairyGUI |
| 程序 / Agent | 按规范生成草稿、跑转换、写业务代码 |
| UI（FairyGUI） | 列表、控制器、动效、九宫等编辑器专属能力 |

关键原则：**图是皮肤，字是数据，能点的是按钮。工具只认「名字」，不认「长得像按钮」。**

---

## 2. 开始前准备

1. 安装 **Figma**（有团队文件编辑权限更佳）
2. 知道本项目画布常用尺寸（例如竖屏 `1080×1920`，以项目约定为准）
3. 转换工具目录（程序已配好）：
   ```
   D:\Project\FairyGUI\psd2fgui
   ```
4. 字体：游戏用到的中文字体，Figma / PS 里尽量一致

---

## 3. 用 Cursor Agent + Figma 生成草稿（可选）

适合：已有参考图，想快速出可改结构的草模。

1. 在 Cursor 对话中说明需求，并附上参考图  
2. **写清交互**（很重要），例如：
   - 「装备格可点击，用 Button」
   - 「金币数字运行时会变，必须是文本」
   - 「背包列表留给 FairyGUI 配 List」
3. Agent 用 Figma MCP 在文件里搭画板  
4. 美术再换正式切图、调版式、核对命名

不要把 Agent 生成的当最终稿直接导：检查命名与文本层是否规范后，再导出 PSD。

---

## 4. Figma 里怎么搭（美术主责）

### 4.1 画板

- 一个界面一个画板  
- **画板名称 = 将来 PSD 文件名**（例如 `RoleEquipScreen`、`VIPScreen`）  
- 不要用空格和奇怪符号（可用英文、数字、下划线）

### 4.2 先想清楚：这块是不是点的？

动手前先问自己一句，再决定组名：

| 这块东西 | 组名必须这样起 | FairyGUI 结果 |
|----------|----------------|---------------|
| **能点击**（装备格、Tab、锻造、加号、返回…） | `Button_xxx` 开头 | 按钮 |
| **勾选** | `CheckButton_xxx` | 勾选按钮 |
| **单选** | `RadioButton_xxx` | 单选按钮 |
| **整块复用摆内容**（里面还可以再塞按钮） | `Com_xxx` 开头 | 普通组件 |
| **只是把图层收到一起**（顶栏、底栏容器） | 随便起名，如 `Header` | 分组（保留层级） |
| **进度条** | `Bar_xxx` 或 `ProgressBar_xxx` | 进度条 |
| **滑动条** | `Slider_xxx` | 滑动条 |
| **标签（标题+图标）** | `Label_xxx` | 标签 |

反例：  
装备格能点，却叫 `Com_EquipSlot` → 会变成普通组件，不是按钮。  
应叫：`Button_EquipSlot`。

### 4.3 组里要拆开的东西

**❌ 不要做：**

- 一张大图上画死了字和按钮外观  
- 把整颗按钮合并成一层像素图（字在图上）

**✅ 要做：**

- 底板 / 图标：图片层  
- 会变的字：**文本层**（不要栅格化）  
- 按钮内部典型结构：

```
Button_Forge
├── bg @up          ← 按钮底图（可再加 @down）
├── icon @icon      ← 需要程序换图标时才加
└── 锻造 @title     ← 按钮上的字，文本层
```

### 4.4 名字里的特殊标记（后缀）

名字里**包含**这些即可（推荐写在后面）：

| 写在名字里 | 用途 |
|------------|------|
| `@up` | 按钮正常态图 |
| `@down` | 按钮按下态图 |
| `@title` | 程序可改的标题字 |
| `@icon` | 程序可换的图（会变成装载器） |
| `@bar` | 进度条横向填充 |
| `@bar_v` | 进度条纵向填充 |
| `@grip` | 滑块把手 |

示例：

- `bg @up`
- `120 @title`
- `coin @icon`
- `fill @bar`

### 4.5 层级（文件夹）

- 用 Figma 的 **Frame / 编组** 整理即可（顶栏、角色区、背包区…）  
- **普通组和 `Group_xxx` 都可以**，转出来都会保留分组  
- 特殊组件（`Button_` / `Com_` …）不受影响，仍按上面表格生成

### 4.6 本地图片怎么放？

1. 准备 PNG（透明底图标、立绘、底板等）  
2. 选中 Figma 里对应的图层 / Frame → 右侧 Fill → Image → 选本地文件  
3. 若运行时要换图，层名要带 `@icon`

### 4.7 Figma 导出前自检

- [ ] 画板名 = PSD 文件名  
- [ ] 能点的都是 `Button_`（不是误写成 `Com_`）  
- [ ] 会变的数字 / 文案都是**文本**，没有栅格化  
- [ ] 按钮里该有的 `@up` / `@title` 已加  
- [ ] 进度条有 `@bar`  
- [ ] 没有「整页拍成一张图」  
- [ ] 红点若只是装饰，可单独小图；不要把字画进红点里

## 5. Figma → PSD

目标：**结构、命名、文本层原样进 PSD**。

建议操作要点：

1. 导出 / 同步为 **PSD**（可用团队约定的插件或官方导出方式；以项目统一工具为准）  
2. 用 Photoshop 打开检查：
   - 组名是否还在（`Button_`、`Com_`、普通组）  
   - 数字、标题是否仍是**文字层**（不是智能对象糊成图）  
3. 若文字变成位图：回到 Figma 修，再导，**不要硬转**

PSD 文件名示例：`RoleEquipScreen.psd`

---

## 6. PSD → FairyGUI（psd2fgui）

程序或 UI 在工具目录执行：

```sh
cd D:\Project\FairyGUI\psd2fgui
npm install
node convert 你的文件路径\RoleEquipScreen.psd
```

常用参数：

```sh
node convert xxx.psd --nopack          # 输出解压目录，方便检查
node convert xxx.psd --ignore-font     # 忽略 PSD 字体名
```

成功后得到 `.fairypackage`（或 `--nopack` 时的文件夹），在 **FairyGUI 编辑器** 中导入。

美术若只需交 PSD，把文件交给程序即可；若自己转，跑上面命令后把包发给 UI/程序。

## 7. FairyGUI 里还要补什么？

工具**不会**自动做好的，需要在编辑器里处理：

| 能力 | 说明 |
|------|------|
| 列表 List | 先做 item（`Com_BagItem` / `Button_BagItem`），再配 GList |
| 控制器 Controller | Tab 切换、显隐状态 |
| 动效 Transition | 打开/关闭动画 |
| 九宫 | 拉伸底板 |
| 3D 装载等 | 项目特殊需求 |

导入后建议检查：

1. 按钮是否都是 Button  
2. 文字是否还能改  
3. 分组层级是否与 Figma 一致  
4. 按项目规范改显示名（如部分包用 `Btn_` / `Text_`，与程序对齐即可）

## 8. 常见错误（踩坑表）

| 现象 | 原因 | 改法 |
|------|------|------|
| 能点的变成了普通组件 | 起名用了 `Com_` | 改成 `Button_` |
| 没有层级、全摊平 | 旧工具或非法命名（应已修复：普通组也会保留） | 确认用的是最新 `psd2fgui` |
| 字不能改 / 字是图 | 文本被栅格化或合并进大图 | 拆成文本层再导 |
| 按钮上的字程序改不了 | 没加 `@title` | 标题层名加 `@title` |
| 图标程序换不了 | 没加 `@icon` | 图标层名加 `@icon` |
| 进度条拖不动进度 | 填充层没 `@bar` | 加上 `@bar` |
| 和参考图差很远 | Agent 草模未换正式图 | 替换 `@icon` 与底板图 |

## 9. 给程序 / Agent 提需求时怎么写？

描述里写清类型，比只发截图更重要。模板：

```
界面名：RoleEquipScreen（竖屏 1080×1920）
参考图：（附上）

交互与类型：
1. 顶栏三个货币：数字可改；旁侧 + 号是 Button
2. 装备格 6 个：可点击 → Button_EquipSlot
3. Tab 四个：Button_Tab_*
4. 锻造 / 分解：Button
5. 背包列表：视觉占位即可，List 在 FairyGUI 配
6. 立绘、装备图标：可换本地图（预留 @icon）

不要底部某某栏（如有）。
```

## 10. 迷你示例：一个可点装备格

```
Button_EquipSlot
├── bg @up              ← 格子底板图
├── icon @icon          ← 装备图标（程序可换）
├── Txt_Tier            ← 文本「1阶」
└── Txt_Level           ← 文本「0级」
```

不要：

```
Com_EquipSlot           ← 错：能点却写成 Com
└── 合并好的整图.png     ← 错：字和图糊在一起
```

## 11. 学习路径（建议半天）

1. 读本文第 4、8 节（搭界面 + 踩坑）  
2. 打开团队 Figma 里已按规范搭的样例（如 `RoleEquipScreen`）对照命名  
3. 自己做一个「只有一个按钮 + 一个数字」的小画板，导出 PSD 转换验证  
4. 再做完整界面

有疑问找程序对一下命名，比导完再返工便宜。
